## 🎨 PROTÓTIPOS VISUAIS (HTML)

### 1. **`app.html`** — A INTERFACE DO APP REAL
**O que é:** A tela do app **logado** — área de membros estilo Netflix.
**Mostra:** sidebar com gêneros, hero rotativo, "continue assistindo" com barra de progresso, "Top 10" com números gigantes, rows por gênero (Mafia Boss, Bilionário, Segunda Chance, Amor Proibido), grid de humores, modal de detalhes com lista de Books.
**Quando mostrar pro parceiro:** "É ASSIM que o app vai ser quando o assinante entrar."

### 2. **`romance.html`** — LANDING PAGE DE VENDA
**O que é:** O site PÚBLICO que vende a assinatura.
**Mostra:** hero com proposta, "como funciona", catálogo amostra, plano único R$ 19,90, comparativo com YouTube Members, roadmap.
**Quando mostrar:** "É ASSIM que vamos converter visitantes em assinantes."

### 3. **`index.html`** — Versão alternativa BÍBLICA
Mantida pra referência caso decida explorar nicho bíblico no futuro. Use `romance.html` como o vigente.

## 🚀 O QUE FAZER AGORA

1. **Abrir `app.html`** no navegador → mostrar como o app vai parecer logado
2. **Abrir `romance.html`** → mostrar como a landing vai vender
3. **Ler `PASSO-A-PASSO-FINAL.md`** → alinhar cronograma e responsabilidades
4. **Decidir entre 2 caminhos técnicos:**
   - **Caminho rápido:** WordPress + plugin Paid Memberships Pro (2-3 semanas pra MVP)
   - **Caminho custom:** Next.js + Postgres + Stripe (6-10 semanas, mais flexível)

Quando decidir, é só me chamar que eu começo o setup técnico de verdade.
